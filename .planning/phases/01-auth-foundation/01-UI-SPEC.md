## Design System — Color Tokens

Tailwind v4 uses CSS custom properties. Define these in `src/index.css`:

```css
@import "tailwindcss";

@layer base {
  :root {
    /* ─── Dark theme (default) ─── */
    --background: 240 10% 3.9%;       /* #09090b  zinc-950  — dominant surface */
    --foreground: 0 0% 98%;           /* #fafafa  zinc-50   */

    --card: 240 10% 3.9%;             /* #09090b  zinc-950  — card matches bg; border differentiates */
    --card-foreground: 0 0% 98%;

    --border: 240 3.7% 15.9%;         /* #27272a  zinc-800  */
    --input: 240 3.7% 15.9%;          /* #27272a  zinc-800  — input background */

    --muted: 240 3.7% 15.9%;          /* #27272a  zinc-800  */
    --muted-foreground: 240 5% 64.9%; /* #a1a1aa  zinc-400  */

    --primary: 217 91% 60%;           /* #3b82f6  blue-500  — CTA / accent */
    --primary-foreground: 0 0% 100%;  /* white text on blue button */

    --secondary: 240 3.7% 15.9%;      /* #27272a  zinc-800  */
    --secondary-foreground: 0 0% 98%;

    --accent: 217 91% 60%;            /* #3b82f6  blue-500  — same as primary */
    --accent-foreground: 0 0% 100%;

    --destructive: 0 72% 51%;         /* #ef4444  red-500   */
    --destructive-foreground: 0 0% 98%;

    --ring: 217 91% 60%;              /* #3b82f6  blue-500  — focus ring */
    --radius: 0.5rem;
  }
}
```

> **Rationale for blue primary:** shadcn neutral default uses white primary (zinc-50) on dark.
> For a server dashboard, blue accent is the industry convention (Portainer, Grafana, k9s web).
> This is a researcher decision — executor may revert to zinc-50 if the developer prefers.

## Typography

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Display | 24px | 700 | 1.2 | `text-2xl font-bold` | "ServerDeck" app name in card header |
| Body | 14px | 400 | 1.5 | `text-sm font-normal` | Muted sub-text, card description |
| Label | 14px | 500 | 1.4 | `text-sm font-medium` | Form field labels |
| Small | 12px | 400 | 1.4 | `text-xs font-normal` | Error messages, helper text |

> **iOS zoom prevention:** Form `<input>` elements MUST use `text-base` (16px) or have
> `font-size: 16px` applied directly. iOS Safari auto-zooms on inputs < 16px.
> shadcn `Input` component defaults to `text-sm` (14px) — override with:
> ```tsx
> <Input className="text-base" ... />
> ```
> This applies to all four login form inputs: Host, Port, Username, Password.

| Dominant (60%) | `#09090b` | `--background` | Full-page background, outer viewport |

**Foreground text:**
- Primary text: `#fafafa` (`--foreground`) — field values, main headings
- Secondary text: `#a1a1aa` (`--muted-foreground`) — card description, placeholder text, port label
- Error text: `#ef4444` (`--destructive`) — error message paragraph only

## Login Page — Full Layout Spec

### Viewport Centering

```tsx
<div className="min-h-svh flex items-center justify-center bg-background px-4">
  {/* Card */}
</div>
```

- `min-h-svh` — uses small viewport height unit, avoids iOS 100vh bug (source: STACK.md)
- `px-4` — 16px horizontal padding on mobile, card doesn't touch screen edges
- No `overflow: hidden` on this wrapper

### Card Dimensions

```tsx
<Card className="w-full max-w-sm">
```

- `max-w-sm` = 384px — fits 390px phone with 4px margin each side
- Card has default shadcn border (`border border-border`) and subtle shadow
- No `backdrop-blur` or glass effect — keep it flat and clean

### Card Header — Branding

```tsx
<CardHeader className="space-y-1 pb-4">
  <div className="flex items-center gap-2">
    <Server size={20} className="text-primary" />
    <CardTitle className="text-2xl font-bold">ServerDeck</CardTitle>
  </div>
  <CardDescription>Connect to your server</CardDescription>
</CardHeader>
```

- `<Server>` icon: 20px, `text-primary` (blue accent) — only branding icon use of accent
- Title: "ServerDeck" — 24px bold, `--foreground`
- Description: "Connect to your server" — 14px, `--muted-foreground`

### Card Content — Form Fields

Field order (vertical stack, `space-y-4`):

1. **Host**
   - Label: "Host"
   - Input: `type="text"`, placeholder `"192.168.1.100"`, `autoCapitalize="none"`, `autoCorrect="off"`, `spellCheck={false}`, `autoComplete="url"`
   - Pre-filled from `localStorage.getItem('sd_host')` on mount

2. **Port**
   - Label: "Port"
   - Input: `type="text"`, `inputMode="numeric"`, `pattern="[0-9]*"`, default value `"22"`, `autoComplete="off"`
   - Pre-filled from `localStorage.getItem('sd_port') ?? "22"` on mount
   - `max-w-24` width override (port numbers are short) — or keep full-width for consistency

3. **Username**
   - Label: "Username"
   - Input: `type="text"`, placeholder `"ubuntu"`, `autoCapitalize="none"`, `autoCorrect="off"`, `spellCheck={false}`, `autoComplete="username"`
   - Pre-filled from `localStorage.getItem('sd_username')` on mount (optional — username stored separately from host)

4. **Password**
   - Label: "Password"
   - Input wrapper: `relative`
   - Input: `type={showPassword ? "text" : "password"}`, NO placeholder, `autoComplete="current-password"`, `pr-11` (right padding for toggle button)
   - Toggle button: absolute positioned `right-0 top-0`, `h-11 w-11` (44×44 touch target), `variant="ghost"`, `type="button"` (prevent form submit)
   - Toggle icon: `<Eye size={16} />` when hidden, `<EyeOff size={16} />` when shown
   - Password is **never** pre-filled or persisted

### Error Message Area

Rendered below all fields, above the submit button. Appears only when login fails.

```tsx
{error && (
  <p className="text-sm text-destructive flex items-center gap-1.5">
    <AlertCircle size={14} />
    {error}
  </p>
)}
```

- Always present in DOM as empty (no layout shift on error appearance) — OR use `min-h-[20px]` placeholder
- Icon: `<AlertCircle size={14} />` from lucide-react (add to inventory)
- Text: `text-sm text-destructive`

### Submit Button

```tsx
<Button
  type="submit"
  className="w-full h-11"
  disabled={isLoading}
>
  {isLoading ? (
    <>
      <Loader2 size={16} className="animate-spin mr-2" />
      Connecting…
    </>
  ) : (
    "Connect"
  )}
</Button>
```

- Full width: `w-full`
- Height: `h-11` (44px — mobile touch target)
- Default: "Connect" — verb-only, direct and clear for this tool
- Loading: `<Loader2 animate-spin>` + "Connecting…" — covers 1-3s SSH handshake
- Disabled during loading: all fields + button disabled (`disabled` prop on each input)
- Background: `--primary` (blue-500) via shadcn default Button

### Card Footer (optional)

Not required for Phase 1. No sign-up link, no "forgot password" — single-user tool.

## Interaction States

### Form States

| State | Visual |
|-------|--------|
| **Default** | Fields empty or pre-filled; no validation decorations |
| **Field focused** | Input border becomes `--ring` (blue-500), 2px ring offset |
| **Field typing** | No inline validation — validate on submit only |
| **Loading** | Button shows spinner + "Connecting…"; all 4 inputs `disabled`; pointer-events none on form |
| **Error** | Button returns to "Connect" (enabled); error paragraph visible in `--destructive`; inputs re-enabled |
| **Success** | Brief moment of "Connect" then `navigate("/")` — no success toast needed |

### Protected Route (Dashboard Stub) States

| State | Visual |
|-------|--------|
| **Resolving auth** | Full-viewport: `min-h-svh flex items-center justify-center` + `<Loader2 size={40} className="animate-spin text-muted-foreground" />` |
| **Authenticated** | Dashboard stub renders |
| **401 received** | `navigate("/login", { replace: true })` — no content flash |

### `/login` Route Guard (already authenticated)

If user navigates to `/login` while already authenticated (valid session cookie):
- Redirect to `/` immediately — no login form shown
- Implemented via: check on mount in the login page component itself using `GET /api/auth/me`

## Copywriting Contract

| Element | Copy |
|---------|------|
| App name | **ServerDeck** |
| Card description | Connect to your server |
| Primary CTA | **Connect** |
| Loading CTA | **Connecting…** |
| Auth error (wrong credentials) | Invalid credentials. Check your host, port, and try again. |
| Network/timeout error | Connection timed out. Verify host and port are reachable. |
| Rate limit error (429) | Too many attempts. Wait a minute and try again. |
| Dashboard stub heading | **ServerDeck** |
| Dashboard stub sub-line | Connected to {host} |
| Logout button | **Log out** |
| Protected route loading | *(spinner only — no text label)* |
| Password show toggle aria-label | Show password |
| Password hide toggle aria-label | Hide password |

> **Error copy notes:**
> - "Invalid credentials" is used for wrong username/password (D-10 — prevents user enumeration)
> - "Connection timed out" is safe to distinguish (reveals nothing about users, only network reachability)
> - Rate limit copy surfaced from 429 response — must be handled in Axios interceptor
> - No destructive confirmation needed for logout (no data loss, idempotent session clear)

## Mobile-Specific Contracts

| Concern | Contract |
|---------|----------|
| **iOS auto-zoom** | All form inputs use `text-base` (16px) to prevent iOS auto-zoom on focus. Override shadcn `Input` default `text-sm` with `className="text-base"`. |
| **Touch targets** | Submit button `h-11` (44px). Password toggle `h-11 w-11` (44×44px). Logout button `h-11`. All meet MOBL-03. |
| **Viewport height** | `min-h-svh` on page wrapper. `svh` = small viewport height, immune to iOS 100vh keyboard bug. |
| **Numeric keyboard** | Port input: `inputMode="numeric"` — triggers number pad on iOS/Android without losing text input |
| **Autocorrect** | Host, Username inputs: `autoCapitalize="none" autoCorrect="off" spellCheck={false}` |
| **Password manager** | Password input: `autoComplete="current-password"` — enables 1Password, Keychain, etc. |
| **Keyboard dismissal** | No special handling needed on login page (no fixed-position elements obscured by keyboard) |
| **Screen width** | Card `max-w-sm` (384px) + `px-4` wrapper = card never wider than 356px on 390px phone. No horizontal scroll. |

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Form labels | All inputs have explicit `<Label htmlFor="...">` wired to `<Input id="...">` |
| Password toggle | `aria-label="Show password"` / `aria-label="Hide password"` + `aria-pressed={showPassword}` |
| Error message | `role="alert"` on error `<p>` — announces to screen reader on appearance |
| Loading button | `aria-busy={isLoading}` on `<Button>` during loading state |
| Form | Standard `<form>` element with `onSubmit` handler; Enter key submits |
| Focus order | Natural DOM order: Host → Port → Username → Password → Connect |
