## Spacing Scale

Standard 8-point scale. All values are multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline dividers |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, header vertical padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

**Mobile-specific exceptions:**
- **Touch target minimum:** `44√ó44px` ‚Äî all interactive elements (buttons, list rows, toolbar keys). This is the Apple HIG / WCAG 2.5.5 minimum; it overrides any component default.
- **Toolbar height:** `calc(44px + env(safe-area-inset-bottom))` ‚Äî already established in CSS var `--toolbar-height`; no change.
- **Terminal header height:** `57px` ‚Äî already established in CSS var `--terminal-header-height`; no change.
- **PWA install banner height:** `48px` fixed ‚Äî fits a single line of text + icon + dismiss button without pushing content.
- **Safe area insets:** All full-bleed elements (toolbar, banner) use `env(safe-area-inset-*)` via `viewport-fit=cover`.

## PWA Install Banner (MOBL-05)

The install banner is a new UI element added to `DashboardPage`. It is the only new visual element in Phase 6.

### Placement

- **Location:** Between the dashboard page header row and the container list ‚Äî immediately below the `<header>` element, above the first container group or empty state.
- **Not:** a floating overlay, bottom sheet, or modal ‚Äî these would obscure content or break the scroll model.

| Property | Value |

| Background | `bg-secondary` (`hsl(var(--secondary))` = zinc-800) |

**Note:** The install button inside the banner is exempt from the 44px rule because it is a supplementary affordance with a large surrounding tap zone (the full 48px banner row). The dismiss button at `h-8 w-8` (32px) is acceptable here as it is not a primary action and the banner itself provides adequate tap area.

### States

| State | Trigger | Behavior |
|-------|---------|---------|
| Hidden (default) | App already running as standalone PWA | `window.matchMedia('(display-mode: standalone)').matches === true` ‚Äî do not render the banner |
| Android prompt available | `beforeinstallprompt` event fired | Show banner; clicking "Install" calls `deferredPrompt.prompt()` |
| iOS Safari hint | iOS UserAgent + not standalone | Show banner with text "Tap Share then 'Add to Home Screen'"; no install button (iOS has no deferred prompt API) |
| Dismissed | User taps ‚úï | Hide banner for the session (no persistence needed ‚Äî a page reload can re-show it) |

### iOS vs Android Branching

```tsx
// Detect iOS Safari (no deferred prompt API)
const isIOS = /iphone|ipad|ipod/i.test(navigator.userAgent)

// isIOS ‚Üí show advisory text only ("Tap Share ‚Üë then Add to Home Screen")
// !isIOS + deferredPrompt available ‚Üí show Install button that calls deferredPrompt.prompt()
// display-mode: standalone ‚Üí render null (already installed)
```

### Animation

- Entrance: none ‚Äî banner appears synchronously when the `beforeinstallprompt` event fires (which happens quickly after page load). No animation needed; a layout jump is acceptable here since the banner appears before the user has scrolled.
- Dismissal: `hidden` class toggle ‚Äî no animation. Consistent with the app's zero-animation philosophy.
