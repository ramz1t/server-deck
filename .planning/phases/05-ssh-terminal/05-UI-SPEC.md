## Interaction Design

### Connection State Machine (D-P5-20, D-P5-21)

```
[mount] → connecting
             │
             ├─── WS open + SSH ready ──→ connected
             │
             └─── WS error / SSH error ──→ failed
                                              │
                                              └─── "Retry" ──→ [unmount + remount] → connecting

[X close / navigate away] → [session terminated, component unmounts]
```

**Connecting state UI:** Terminal container area shows a centered column: `<Loader2 className="h-6 w-6 animate-spin text-zinc-400" />` + `<p className="text-zinc-400 text-sm mt-2">Connecting…</p>`. Background remains `#09090b`. xterm DOM is rendered but hidden (`opacity-0`) to allow FitAddon pre-calculation.

**Failed state UI:** Replace spinner with an inline error block (no full-page takeover):
```
<div className="flex flex-col items-center gap-3 text-center p-6">
  <AlertCircle className="h-8 w-8 text-red-400" />
  <p className="font-semibold text-red-400">Connection failed</p>
  <p className="text-sm text-zinc-400">{error.message}</p>
  <Button variant="outline" size="sm" className="h-11" onClick={onRetry}>Retry</Button>
</div>
```

### Ctrl Modifier (D-P5-11)

1. **Tap Ctrl once** → `ctrlActive = true`. Ctrl button renders with active style (blue ring). No character sent.
2. **Tap a letter key (keyboard)** → send `\x01`–`\x1a` (A=1…Z=26). Reset `ctrlActive = false`.
3. **Tap Ctrl again while active** → `ctrlActive = false`. Nothing sent.
4. **Tap another toolbar button while Ctrl active** → send Ctrl+that-character's sequence if applicable; always reset `ctrlActive = false`.
5. **Ctrl+C** → `\x03`, **Ctrl+D** → `\x04`, **Ctrl+L** → `\x0c`, **Ctrl+Z** → `\x1a`.

The Ctrl button does NOT interact with normal keyboard letter input from the on-screen keyboard — it only modifies the next toolbar button tap (toolbar is self-contained).

### Toolbar Key Sequences (D-P5-10, D-P5-12)

| Button label | Sequence sent | Notes |
|---|---|---|
| `Ctrl` | — | Modifier only, toggles active state |
| `Tab` | `\t` (0x09) | |
| `Esc` | `\x1b` | |
| `↑` | `\x1b[A` | ANSI cursor up |
| `↓` | `\x1b[B` | ANSI cursor down |
| `←` | `\x1b[D` | ANSI cursor left |
| `→` | `\x1b[C` | ANSI cursor right |
| `\|` | `\x7c` | Pipe character |
| `` ` `` | `\x60` | Backtick |
| `~` | `\x7e` | Tilde |
| `/` | `\x2f` | Forward slash |

All sequences are sent via the xterm.js `Terminal.input(sequence)` method AND via the WebSocket `socket.send(sequence)` — AttachAddon handles the WS direction, but direct `terminal.input()` ensures immediate local echo where needed.

### Resize Handling (D-P5-17, D-P5-23)

1. `ResizeObserver` watches the terminal container div.
2. On resize: `requestAnimationFrame(() => fitAddon.fit())`.
3. After `fitAddon.fit()`: read `terminal.cols` + `terminal.rows`, send `JSON.stringify({ type: 'resize', cols, rows })` via WebSocket.
4. Initial fit: run on `connected` state transition, inside `requestAnimationFrame`.

### Session Teardown on Unmount (D-P5-25)

```typescript
useEffect(() => {
  return () => {
    terminal.dispose()     // prevents WebGL context exhaustion (iOS)
    socket.close()         // triggers server WS close → stream.destroy() + conn.end()
  }
}, [])
```

## Mobile-Specific Rules

| Rule | Specification | Source |
|------|--------------|--------|
| Viewport unit | Use `dvh` (dynamic viewport height), not `vh` or `svh` | D-P5-05, iOS keyboard handling |
| Safe area padding | Toolbar: `padding-bottom: env(safe-area-inset-bottom)` | D-P5-08 |
| Minimum tap target | All toolbar buttons and header buttons: `min-h-[44px] min-w-[44px]` | MOBL-03 |
| Autocorrect disabled | Terminal container div: `autoCorrect="off" autoCapitalize="off" spellCheck={false} data-gramm="false"` | D-P5-26, MOBL-04 |
| Scroll prevention | Terminal container: `touch-action: none` via Tailwind `touch-none` to prevent page scroll conflict | iOS Safari |
| Font smoothing | `-webkit-font-smoothing: antialiased` on terminal container | iOS rendering |
| Toolbar scroll | `overflow-x: auto; -webkit-overflow-scrolling: touch; scrollbar-width: none` — toolbar scrolls horizontally on < 375px screens | Discretion: agent |
| Keyboard avoidance | `height: calc(100dvh - ...)` recalculates when iOS keyboard appears — no JS workaround needed with `dvh` | D-P5-05 |
| No zoom on focus | Header buttons use `h-9` (36px) but terminal focus does not trigger 16px font-size zoom because xterm manages its own input model | xterm.js behavior |

## xterm.js Integration Notes (D-P5-22 through D-P5-26)

### Package Requirements

```
@xterm/xterm          — core terminal
@xterm/addon-fit      — FitAddon (responsive sizing)
@xterm/addon-attach   — AttachAddon (WS bidirectional pipe)
```

### Mount Sequence

```
1. Create Terminal instance with XTERM_OPTIONS
2. Create FitAddon, loadAddon(fitAddon)
3. terminal.open(containerRef.current)
4. Set terminal container opacity-0 during connecting state
5. On connected:
   a. Create WebSocket to /api/terminal
   b. ws.onopen → create AttachAddon(ws), terminal.loadAddon(attachAddon)
   c. requestAnimationFrame(() => fitAddon.fit())
   d. Send initial resize message
   e. Set container opacity-1
6. Set up ResizeObserver → requestAnimationFrame(() => fitAddon.fit()) + send resize message
```

### Unmount Sequence (D-P5-25)

```
1. resizeObserver.disconnect()
2. attachAddon.dispose()
3. terminal.dispose()    ← MUST be last; prevents WebGL context leak on iOS
4. ws.close()
```

### WS Message Protocol (D-P5-17)

- **Client → Server (resize):** `JSON.stringify({ type: 'resize', cols: number, rows: number })`
- **Client → Server (input):** raw string/binary (handled by AttachAddon)
- **Server → Client (output):** raw PTY output bytes (handled by AttachAddon)

Server disambiguates via `JSON.parse` try/catch — if parsed and `data.type === 'resize'`, call `stream.setWindow()`; otherwise pipe raw to PTY stdin.
